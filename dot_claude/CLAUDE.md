# User Instructions

## Workflow selection

Choose the lightest workflow that can produce a correct result. Superpowers and Ultracode are available, but neither is mandatory for every task.

Use Superpowers when the work benefits from its software-development methodology:

- ambiguous feature work that needs requirements, design, or a plan before coding
- multi-step implementation where TDD, review, or worktree discipline reduces risk
- debugging that needs systematic root-cause analysis instead of guessing
- PR/review/branch-completion work where the Superpowers lifecycle fits
- any task where the user explicitly asks for Superpowers

Use Ultracode when the work needs high-effort dynamic orchestration:

- broad technical research, architecture, migration, or tradeoff analysis
- high-uncertainty or high-impact decisions that need independent critique
- large refactors, audits, or investigations where multiple passes reduce blind spots
- tasks that should synthesize evidence from several sources or agents
- any task where the user explicitly asks for Ultracode

Use both when appropriate: Ultracode can drive deep investigation or adversarial review, while Superpowers can structure the implementation workflow.

Do not use either workflow for small direct edits, simple commits, quick answers, status checks, or obvious single-step fixes. In those cases, act directly.

When starting substantial work, state the chosen workflow in one sentence and why. If the choice is borderline, make a reasonable call and proceed; only ask the user when the workflow choice would materially change time, cost, risk, or scope.
